# 创建项目文档

当用户说"创建项目文档"时，执行以下步骤：

## 步骤

1. **扫描项目结构**
   - 列出项目根目录的文件和子目录
   - 识别主要入口文件（如 app.py, main.py, index.html 等）
   - 识别技术栈（通过 requirements.txt, package.json 等）
   - 识别核心模块和代码结构

2. **阅读关键文件**
   - 查看入口文件的 outline
   - 查看配置文件
   - 查看已有的 README.md 或其他文档

3. **生成 `GEMINI.md`（项目级）**
   在项目根目录创建或更新 `GEMINI.md`，包含以下章节：

   ```markdown
   # 项目名称

   ## 项目概述
   简要说明项目是什么、解决什么问题

   ## 技术栈
   列出后端、前端、数据库等技术选型

   ## 项目结构
   核心目录和文件说明（树形结构）

   ## 核心模块
   每个关键模块的功能说明

   ## 开发约定
   命名规范、代码风格、分支策略等（如有）

   ## 当前进度
   - ✅ 已完成的功能
   - 🔄 进行中
   - 📌 待办

   ## 已知问题
   列出已知的 bug 或技术债务
   ```

4. **确认并反馈**
   - 把生成的文档展示给用户
   - 询问是否需要补充或修改
